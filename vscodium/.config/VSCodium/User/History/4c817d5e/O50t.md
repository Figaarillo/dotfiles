### `src/router/router.ts`

Es una clase generica que se encargaria de servir como base, para las demas rutas en donde realiza ciertas opoeraciones básicas
Entre otras cosas, en la clase se espera que el génerico recibido sea un controllador.
A su vez, la clase tiene un metodo routes, que se espera que sea 'modificado' o implementado por quien la herede para que luego se ejecute en el constructor de la clase base.

#### Cosas importantes

- **Clase**

  ```ts
  class BaseRouter<T> {
    public router: Router
    public controller: T

    constructor(Controller: { new (): T }) {
      this.router = Router()
      this.controller = new Controller()
      this.routes()
    }
    routes() {}
  }
  ```

- **Ejecución de las rutas**

  ```ts
  public routes(): Array<Router> {
      return [new UserRouter().router]
  }
  ```

## Git Hooks

Para intentar asegurar un estandar dentro del proyecto, ya sea de estilos, mensajes en los commits entre otros, se hará uso de los git Hook. Estos, previo a que se haga un commit o un push van a ejecutar un script. Los script que ejecutaran, estan enfocados en los estilos, compilar el código y en generar un estandar para los mensajes de commit

### [Husky](https://typicode.github.io/husky/)

¿Qué es Hustky? Husky es una herramienta que permite configurar ganchos (hooks) de Git para ejecutar scripts automatizados en ciertos momentos, como antes de realizar un commit.

Para poder usarlo corremos el siguiente comando

```bash
pnpm dlx husky-init && pnpm install
```

Esto va a:

1. Añadir un script `prepare` en el `package.json`
2. Crear un `pre-commit` de plantilla que puede ser editado
3. Configurar la ruta de Git hooks

### [lint-staged](https://www.npmjs.com/package/lint-staged#Configuration)

Lint staged es basicamente un hook que ejecurá el linter. Para este proyecto esta configurado par que se ejecute antes de un commit

### [commitlitn](https://commitlint.js.org/#/)

Commitlint es un hook que verificará que los mensajes de commit respeten las [convenciones de los commits](https://www.conventionalcommits.org/es/v1.0.0/)

## Pre-commit

Antes de la confimación de un commit, se ejecutará un hook que se encarga de relizar ciertos crequeos.
El fichero de husky tiene establecido que se llame al script pre-commit

```bash
pnpm pre-commit
```

De esta forma se le indica que se tiene que ejecutar lo que contenga el fichero lint-staged (`.lintstagedrc`)
Este contiene un scritp que indica lo que se dede ejecurtar

```bash
pnpm run pre-check
```

Este scritp se encarga de ejecuar lo siguiente

```bash
pnpm run ts-check && pnpm run prettier && pnpm run lint
```

## Docker

configuración de docker-compose

```yml
version: '3.1'

services:
  app_db:
    image: mysql:5.7
    restart: always
    environment:
      MYSQL_DATABASE: app_db
      MYSQL_ROOT_USER: admin
      MYSQL_ROOT_PASSWORD: secret
      MYSQL_USER: figarillo
      MYSQL_PASSWORD: secret
    volumes:
      - ./mysql:/docker-entrypoint-initdb.d
      - ./my.cnf:/etc/mysql/my.cnf
    command: --init-file /docker-entrypoint-initdb.d/init.sql
    ports:
      - '3306:3306'
```

Este archivo de Docker Compose define un servicio llamado `app_db` que utiliza la imagen mysql:5.7.
Crea una base de datos llamada "app_db" y configura el usuario raíz como "admin" con contraseña "secret".
También crea un usuario adicional llamado "figarillo" con la contraseña "secret".
Monta dos volúmenes, uno para ejecutar archivos SQL de inicialización y otro para la configuración personalizada de MySQL.
El servicio expone el puerto 3306 para acceder al servidor MySQL.
